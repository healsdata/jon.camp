Peter LaVigne argues for shifting from reviewing AI-generated code (reading it line by line) to verifying it through property-based tests, mutation testing, and side-effect constraints. The idea is to narrow the solution space enough that passing code is almost certainly correct code. He suggests treating generated code like compiled output rather than something that needs to be readable or maintainable.
